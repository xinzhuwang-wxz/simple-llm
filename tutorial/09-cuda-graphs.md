# 3.3 CUDA Graph优化

> 📍 **问题**：每次GPU操作都需要CPU提交，这会有什么性能开销？如何优化？

---

## 3.3.1 内核启动开销

### 💡 概念解释

每次GPU操作都需要：
1. CPU准备操作参数
2. 提交到GPU驱动
3. GPU调度执行

这有**固定开销**，小操作多时累积很大。

---

## 3.3.2 CUDA Graph原理

### 📝 工作流程

```
传统方式:
┌─────────────────────────────────────────────────────────┐
│ Step 1: CPU提交 op1 → GPU执行                           │
│ Step 2: CPU提交 op2 → GPU执行                           │
│ Step 3: CPU提交 op3 → GPU执行                           │
│ ...每次都有CPU-GPU通信开销                               │
└─────────────────────────────────────────────────────────┘

CUDA Graph:
┌─────────────────────────────────────────────────────────┐
│ 1. 捕获: 运行一次完整流程，记录所有操作                    │
│ 2. 优化: 一次性提交整个图                                │
│ 3. 重放: 后续直接重放图，无需CPU参与                      │
└─────────────────────────────────────────────────────────┘
```

## 3.3.3 SimpleLLM实现

### 📝 代码位置

`llm.py` - `_capture_cuda_graph` 方法

```python
# 简化版CUDA Graph实现
graph = torch.cuda.CUDAGraph()
with torch.cuda.graph(graph):
    # 在这里执行一次Decode
    # 这次执行会记录所有操作
    self._decode_step_captured(input_ids, ...)

# 后续直接重放
graph.replay()  # 比普通调用快
```

## 3.3.4 本节小结

### ✅ 关键要点

1. **问题**：每次GPU操作有启动开销
2. **解决**：CUDA Graph预捕获计算图
3. **效果**：消除开销，提升性能
