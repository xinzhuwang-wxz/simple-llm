# 3.2 Paged Attention与内存管理

> 📍 **问题**：传统的KV Cache管理方式有什么问题？如何更高效地管理内存？

---

## 3.2.1 传统KV Cache的问题

### 💡 概念解释

传统方式使用**连续内存**存储KV Cache：

```
传统方式的问题:
┌─────────────────────────────────────────────────────────────┐
│ 预分配最大长度（如4096）的连续内存                            │
│ ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐        │
│ │Slot0│Slot1│Slot2│Slot3│Slot4│Slot5│ ... │ ... │        │
│ │100tok│50tok│200tok│80tok│ ... │ ... │     │     │        │
│ └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘        │
│                                                              │
│ 问题:                                                       │
│ 1. 实际只用100 tokens，却占用4096的空间                      │
│ 2. 一个序列完成后，空出的空间难以利用                         │
│ 3. 内存碎片化                                               │
└─────────────────────────────────────────────────────────────┘
```

---

## 3.2.2 Paged Attention的解决方案

### 💡 概念解释

**Paged Attention**借鉴操作系统的虚拟内存和分页机制：

```
Paged Attention:
┌─────────────────────────────────────────────────────────────┐
│ 将KV Cache分成固定大小的"页"（如16 tokens）                  │
│                                                              │
│ 序列A: [页0] [页1] [页2] [页3]  (不连续)                   │
│ 序列B: [页0] [页1]                                         │
│ 序列C: [页0] [页1] [页2] [页3] [页4] [页5]                 │
│                                                              │
│ 优势:                                                       │
│ 1. 按需分配页，需要多少分配多少                              │
│ 2. 序列完成时，立即释放所有页                                │
│ 3. 不同序列的页可以交错存放                                  │
└─────────────────────────────────────────────────────────────┘
```

```python
# KV Cache: [num_slots, 2, max_seq_len, num_heads, head_dim]

```
