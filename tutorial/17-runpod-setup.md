# 5.2 RunPod租卡完整指南

> 📍 **问题**：我决定使用RunPod，具体应该怎么租卡？请给出完整的步骤。

---

## 5.2.1 注册账号

### 📝 步骤1：访问RunPod

```
1. 打开浏览器，访问 https://runpod.io
2. 点击页面右上角的 "Sign Up" 或 "注册"
```

### 📝 步骤2：填写注册信息

| 字段 | 说明 |
|------|------|
| Email | 使用你的邮箱 |
| Password | 设置密码（建议使用密码管理器） |
| Username | 选择一个用户名 |

### 📝 步骤3：验证邮箱

- 打开注册邮箱
- 点击RunPod发送的验证链接
- 完成验证

---

## 5.2.2 充值账户

### 📝 问题

**问题**：RunPod支持哪些支付方式？

**答案**：

| 支付方式 | 说明 |
|----------|------|
| **信用卡** | 最常用，Visa/MasterCard |
| **PayPal** | 如果有PayPal账户 |
| **加密货币** | 部分支持 |

### 📝 充值步骤

```
1. 登录RunPod账号
2. 点击右上角头像 → "Billing" 或 "账户"
3. 点击 "Add Funds" 或 "充值"
4. 输入充值金额（建议 $20-50 起步）
5. 选择支付方式并完成支付
```

### ⚠️ 注意事项

- **最低充值**：$10
- **建议充值**：$30-50（够用几次实验）
- **计费方式**：按小时计费，停止实例后不再收费

---

## 5.2.3 查找和选择GPU

### 📝 步骤1：进入Pod市场

```
1. 登录后，点击 "Pod" 或 "GPU Cloud"
2. 进入 "Browse Templates" 或 "Find GPUs"
```

### 📝 步骤2：筛选GPU

在搜索/筛选选项中：

| 设置 | 推荐值 |
|------|--------|
| **GPU Type** | NVIDIA H100 |
| **GPU Count** | 1 |
| **Memory** | 80GB（如果可用） |

### ⚠️ 重要提示

**SimpleLLM需要H100 80GB！**

如果你看不到H100：
1. 检查是否有地区限制
2. 尝试不同时间段（热门时段可能没货）
3. 考虑A100作为备选（但性能会差一些）

---

## 5.2.4 配置实例

### 📝 问题

**问题**：如何配置实例？

**答案**：

```
1. 选择GPU后，点击 "Deploy" 或 "部署"

2. 基本配置：
   - GPU: NVIDIA H100 80GB ✅
   - GPU Count: 1
   - Instance Type: Single GPU

3. 存储配置：
   - System Storage: 至少 50GB（建议100GB）
   - 如果需要更多空间，可以添加额外存储

4. 镜像选择（重要！）：
   - 方式1：使用社区模板
     * Template: 搜索 "PyTorch 2.9"
     * 选择包含 CUDA 12.8 的镜像
   - 方式2：自定义镜像
     * 选择基础镜像如 ubuntu 22.04
     * 后续自行安装环境
```

### 📝 推荐配置

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| **GPU** | H100 80GB | 必须 |
| **Instance** | Single GPU | 单卡足够 |
| **存储** | 100GB | 模型文件需要空间 |
| **镜像** | pytorch/torchserve:2.9.1-cuda12.8 | 推荐 |
| **端口** | 默认即可 | 不需要额外配置 |

---

## 5.2.5 启动实例

### 📝 步骤

```
1. 确认配置无误后，点击 "Deploy" 或 "Deploy Now"

2. 等待实例创建（约1-3分钟）

3. 状态变为 "Running" 后，可以连接
```

### 📝 查看连接信息

实例启动后，你会看到：

```
==========================================
Your Pod is Ready!
==========================================

SSH Connection:
ssh root@<your-ip-address>

RunPod Terminal:
https://my.runpod.io/terminal/<some-id>

==========================================
```

---

## 5.2.6 连接到实例

### 📝 方式1：使用RunPod终端（推荐新手）

```
1. 在实例页面，点击 "Connect" 或 "连接"
2. 选择 "RunPod Terminal"
3. 浏览器会打开一个终端窗口
4. 可以直接执行命令
```

### 📝 方式2：使用本地SSH

```bash
# 打开终端，执行：
ssh root@<your-ip-address>

# 首次连接可能需要确认，输入 yes
# 然后输入密码（如果设置了）
```

### ⚠️ 注意事项

- 如果SSH连接失败，检查实例状态是否为"Running"
- 检查安全组/防火墙设置
- RunPod终端是最简单的连接方式

---

## 5.2.7 验证GPU可用

### 📝 问题

**问题**：如何确认GPU正常工作？

**答案**：

在终端中执行：

```bash
# 查看GPU信息
nvidia-smi

# 或者使用PyTorch验证
python -c "import torch; print(f'CUDA available: {torch.cuda.is_available()}'); print(f'GPU: {torch.cuda.get_device_name(0)}')"
```

### 预期输出

```
CUDA available: True
GPU: NVIDIA H100 80GB
```

---

## 5.2.8 成本控制建议

### 📝 问题

**问题**：如何控制成本，避免意外收费？

**答案**：

| 建议 | 说明 |
|------|------|
| **设置使用限额** | 在Billing中设置最大支出限额 |
| **用完即停** | 实验结束后停止实例 |
| **使用Spot实例** | 更便宜（但可能被抢占） |
| **监控使用** | 定期查看Billing页面 |

### ⚠️ 重要提醒

- RunPod是**按小时计费**，实例运行就会收费
- **停止**实例不收费，但会保留存储
- **删除**实例会删除所有数据

---

## 5.2.9 本节小结

### ✅ 关键要点

1. **注册账号**：访问runpod.io，用邮箱注册
2. **充值**：建议$20-50起步
3. **选择GPU**：必须是H100 80GB
4. **配置镜像**：选择PyTorch 2.9 + CUDA 12.8
5. **连接**：使用RunPod终端或SSH
6. **验证**：用nvidia-smi确认GPU可用
